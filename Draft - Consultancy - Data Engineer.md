# Data Engineer

The Data Engineer is the primary technical delivery role across the Platform, Build, and Release phases, responsible for building and configuring the data solution in line with the agreed Design. This role contributes technical input during the Definition stage to inform platform and data architecture decisions, and supports the Reporting and Quality Assurance phases by resolving data and pipeline issues as they arise. On larger projects, multiple Data Engineers may be assigned, with the Data Lead coordinating across them. The Data Engineer role encompasses seniority levels from Data Engineer through to Senior Data Engineer. More senior engineers will typically take ownership of more complex workloads and broader areas of the solution, while less experienced engineers may focus on specific components under the oversight of the Data Lead.

**Framework Involvement**

Discovery 
- Contributes technical input across platform, infrastructure, and data architecture subjects during Platform Discovery and Source System Discovery.
- Provides technical input on ingestion methods, connectivity, and data profiling to inform source system and data architecture design decisions.
- Contributes to the DevOps subject within Operating Model Discovery where assigned, providing technical input on existing deployment practices, source control, and environment strategy.

Design
- Contributes technical input across platform, data architecture, and infrastructure subjects, working alongside the Data Consultant depending on the subject area.
- Contributes to the production of Design outputs related to platform configuration, environment strategy, deployment pipelines, and ingestion design.
- Contributes to source to target mappings and data lineage documentation.

Platform 
- Leads the technical setup and configuration of the data platform in line with the agreed Design, including provisioning capacities, workspaces, and domains.
- Establishes environments and configures deployment pipelines and source control in line with the agreed environment strategy.
- Implements roles and access controls across all layers and environments.
- Configures logging, monitoring, and alerting in line with the agreed operational model.
- Confirms and tests connectivity to all source systems required for ingestion.
- Builds reusable components and accelerators to support consistent and efficient development during the Build phase.
- Produces the platform readiness note for review and sign-off by the Data Lead.

Build
- Builds ingestion pipelines for each source entity, handling inserts, updates, deletes, and error conditions in line with the agreed design.
- Develops data transformations across each layer of the solution, from ingestion through to the semantic layer, applying the business logic and modelling approach confirmed during Design.
- Applies data quality rules and any master data management or probabilistic matching logic confirmed during Definition.
- Validates transformation logic and business rules with the customer's data owners and subject matter experts throughout the phase.
- Conducts unit testing and peer review throughout, documenting outcomes and tracking any issues identified for resolution before the phase closes.
- Leads the technical execution of data migration where migration is in scope, working alongside the Data Consultant who leads the discovery and design of the migration approach.
- Produces the Build completion note for review and sign-off by the Data Lead.

Reporting
- Resolves data and pipeline issues that surface during report build, ensuring the data layer remains stable and accurate throughout the Reporting phase.

Quality Assurance
- Resolves defects identified in the data and pipeline layers during testing, documenting outcomes and confirming resolution before the phase closes.

Release
- Executes the production deployment in line with the agreed deployment plan, under the oversight of the Data Lead.
- Validates that production data meets expectations immediately following release and resolves any data or pipeline issues identified during the initial post-release period.
