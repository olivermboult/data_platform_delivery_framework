# I&S Engineer

The I&S Engineer is a specialist role assigned to projects where the scope includes complex security or governance requirements. This role delivers infrastructure and security sessions from Discovery onwards, leads the production of all infrastructure and security design outputs during Design, and works alongside the Data Engineer during the Platform phase to configure security and access controls in line with the agreed design. The role has no involvement on projects where infrastructure and security complexity does not warrant their assignment. 

The I&S Engineer role encompasses seniority levels from I&S Engineer through to Senior I&S Engineer. More senior engineers will typically take ownership of more complex security and governance workloads and broader areas of the solution, while less experienced engineers may focus on specific components under the oversight of the Data Lead.

**Framework Involvement**

Pre-Discovery
- Contributes consultatively to Platform, Data and AI Initiation sessions where assigned, providing specialist input across infrastructure and security subjects including software provider selection, data protection, and capacity management.

Discovery
- Delivers infrastructure and security sessions across all relevant subjects during Platform Discovery, including roles and access, data protection, audit and compliance, logging and monitoring, environment strategy, and disaster recovery.
- Produces initial outputs from infrastructure and security sessions for review by the Data Lead.
- Contributes specialist infrastructure and security input to Source System Discovery where data protection and access requirements affect source system design decisions.
- Contributes to the DevOps subject within Operating Model Discovery where assigned, providing specialist input on security and governance considerations within the customer's existing deployment practices.

Design
- Leads the production of Design outputs across all infrastructure and security subjects, including data protection design, roles and access design, audit and compliance framework, logging and monitoring architecture, environment strategy, and disaster recovery design.
- Works alongside the Data Consultant and Data Engineer to ensure infrastructure and security design decisions are consistent with the broader Design.
- Confirms that all infrastructure and security design outputs have been reviewed and approved by the relevant customer stakeholders before build begins.

Platform
- Works alongside the Data Engineer during the Platform phase, contributing specialist expertise across security and access configuration, data protection controls, logging and monitoring setup, and environment strategy implementation.
- Confirms that all security and access controls are implemented in line with the agreed design before the platform readiness note is produced.

Quality Assurance
- Resolves defects identified in security and access controls during testing, documenting outcomes and confirming resolution before the phase closes.

Release
- Validates that access controls and security configuration are correctly in place in the production environment following deployment and resolves any issues identified during the initial post-release period.
