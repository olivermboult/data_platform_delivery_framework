# Reporting Platform

## Review the Existing Reporting Platform

**Activities**
- Identify whether the customer has an existing reporting tool in use and document whether it will be replaced, retained, or supplemented as part of the project.
- Document the name, version, and licensing model of any reporting tools currently in use.
- Identify any known limitations or pain points with the existing tooling that the new solution should address.

**Outcomes**
- Documented existing reporting tool in use and its expected status in the new solution.
- Documented name, version, and licensing model of reporting tools currently in use.
- Documented known limitations and pain points with the existing tooling.

**Pre-requisites**
- A stakeholder with knowledge of the existing reporting tool estate is available for a workshop.

**Basis of Estimate**
- Number of reporting tools currently in use
- Extent of known limitations or pain points to document

## Establish the Reporting Platform Selection Position

**Activities**
- Identify whether the reporting tool has already been selected, and if so document the tool and the basis for the decision.
- Identify whether the choice is open to recommendation, and if so capture the customer's key requirements and priorities to inform the evaluation.
- Identify whether the preferred or selected reporting tool is compatible with the data platform, and document any known integration constraints.
- Identify any internal IT or procurement policies that may affect which reporting tools can be considered.
- Capture the licensing model for the reporting tool and identify any cost implications for the project or for ongoing operations.

**Outcomes**
- Documented reporting tool selection position, including the tool chosen and basis for the decision where a choice has been made.
- Documented customer requirements and priorities to inform tool evaluation where selection remains open.
- Documented compatibility of the reporting tool with the data platform and any known integration constraints.
- Documented IT or procurement policies that may affect which tools can be considered.
- Documented reporting tool licensing model, including cost implications for the project and ongoing operations.

**Pre-requisites**
- Outputs from the Review the Existing Reporting Platform task are available.

**Basis of Estimate**
- Whether the tool has been selected or an evaluation is required
- Complexity of compatibility or integration requirements
- Number of IT or procurement policies to review

## Identify Platform Constraints and Considerations

**Activities**
- Identify any feature limitations or capability gaps in the reporting tool that may affect the design.
- Identify whether the internal team and report creators are already familiar with the reporting tool, and document any learning curve that may affect the project timeline.
- Identify whether any additional licences or tooling will need to be procured, and document the process and expected timeline for doing so.

**Outcomes**
- Documented feature limitations or capability gaps in the reporting tool that may affect the design.
- Documented familiarity of the internal team and report creators with the reporting tool, and any learning curve that may affect the project timeline.
- Documented additional licensing or tooling procurement requirements, including the process and expected timeline.

**Pre-requisites**
- Outputs from the Establish the Reporting Platform Selection Position task are available.

**Basis of Estimate**
- Complexity of feature gaps or capability limitations
- Extent of team familiarity with the reporting tool
- Whether additional licensing procurement is required
